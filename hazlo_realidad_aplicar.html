Hazlo realidad (Aplicar)

¡Es hora de poner en práctica lo que has aprendido y diseñado! Ahora, llevarás tu red del papel a la simulación virtual o al montaje práctico para verificar su funcionamiento. Sigue estos pasos:

1. Construye tu red

Opción 1: Crear la red físicamente o usar un simulador avanzado:

Usa herramientas de simulación para crear y probar tu diseño:

  1. Simulación virtual (modelo preferente de continuación de la situación de aprendizaje):
    • Utiliza Packet Tracer o cualquier otro simulador recomendado, como GNS3 o EVE-NG.
    • Diseña la red según tu esquema, añadiendo cables, routers, switches y puntos de acceso.
    • Configura los dispositivos con las direcciones IP y reglas necesarias.
    • Existen múltiples simuladores, aunque el más avanzado es CISCO Packet Tracer, si decides usaro, visualiza antes esta lista de reproducción de vídeos en Youtube del profesor Eliezer De León.
  2. Montaje práctico (si cuentas con materiales en el aula-taller) (se asume que esta modalidad de continuar la situación de aprendizaje no va a llevarse a cabo):
    • Conecta los cables físicamente.
    • Configura los routers y switches siguiendo tu diseño.
    • Verifica que los puntos de acceso inalámbricos funcionen correctamente.

Opción 2: Diseñar la red usando un simulador/diagramizador básico y documentar el proceso de configuración de los componentes:

Utilizarás una herramienta básica como Draw.io o Lucidchart para realizar el diseño de tu red. Estas herramientas permiten crear diagramas de forma intuitiva y con una apariencia profesional.

  1. Abre la herramienta:
    1. Accede a Draw.io (https://app.diagrams.net/) o Lucidchart (https://www.lucidchart.com/).
    2. Si no tienes cuenta, crea una gratuita.
  2. Crea el diseño de tu red:
    1. Usa iconos representativos para cada componente: routers, switches, puntos de acceso, paneles de parcheo, servidores y dispositivos finales.
    2. Representa las conexiones utilizando líneas que diferencien entre cableado UTP, fibra óptica o enlaces inalámbricos.
  3. Detalles clave:
    1. Incluye etiquetas para identificar cada dispositivo.
    2. Asegúrate de representar correctamente las áreas clave según el espacio asignado (estadio, sala de prensa, fan zone o VIP).
  4. Exporta y guarda tu diseño:
    1. Una vez terminado, exporta el diagrama en formato PDF o imagen para incluirlo en tu informe final.

Ejemplo de topología lógica de una red LAN:

1. Packet Tracer

Packet Tracer de Cisco es un software de simulación de redes que permite a los estudiantes probar cómo funciona la red y responder preguntas del estilo «¿qué pasaría si...?». Una de las grandes ventajas de usar este software es que permite "ver" (opción "Simulation") cómo se mueven los paquetes entre los diferentes dispositivos (switches, routers, PCs), además de poder examinar rápidamente el contenido de cada uno en las distintas "capas" y "datos".

2. GNS3 (Graphical Network Simulator 3)

  • Descripción: Un emulador de redes avanzado que permite simular dispositivos reales utilizando imágenes de sistemas operativos como Cisco IOS, Juniper, etc.
  • Ventajas:
    • Muy flexible y compatible con una amplia gama de dispositivos.
    • Permite combinar dispositivos físicos con virtuales.
    • Excelente para usuarios avanzados y quienes deseen trabajar en redes reales.
  • Uso ideal: Proyectos más avanzados que requieren configuraciones reales de hardware.

3. EVE-NG (Emulated Virtual Environment Next Generation)

  • Descripción: Una plataforma potente para emular redes virtuales completas, muy utilizada en entornos educativos y profesionales.
  • Ventajas:
    • Interfaz gráfica web que facilita la creación de topologías.
    • Compatible con una gran variedad de imágenes de fabricantes como Cisco, Juniper, Fortinet, Palo Alto, etc.
    • Permite la colaboración en tiempo real.
  • Uso ideal: Entrenamiento en redes empresariales y certificaciones avanzadas.

4. Cisco VIRL (Virtual Internet Routing Lab)

  • Descripción: Simulador oficial de Cisco que permite probar y configurar redes con dispositivos virtuales de la marca.
  • Ventajas:
    • Incluye imágenes oficiales de Cisco IOS.
    • Ideal para practicar configuraciones para certificaciones Cisco (CCNA, CCNP, etc.).
  • Desventajas: Es una herramienta de pago.
  • Uso ideal: Usuarios avanzados y aquellos enfocados en dispositivos Cisco.

5. NetSim (Boson NetSim)

  • Descripción: Un simulador de pago diseñado específicamente para la práctica de certificaciones Cisco.
  • Ventajas:
    • Interfaz amigable y fácil de usar.
    • Incluye laboratorios guiados para certificaciones como CCNA y CCNP.
  • Desventajas: No es tan flexible como GNS3 o EVE-NG.
  • Uso ideal: Estudiantes que preparan certificaciones de Cisco.

6. iNetSim (Internet Network Simulator)

  • Descripción: Una herramienta más ligera diseñada para simular redes pequeñas, ideal para prácticas básicas.
  • Ventajas:
    • Fácil de usar y con menos requerimientos técnicos.
    • Ideal para principiantes.
  • Uso ideal: Introducción a redes y prácticas en entornos simples.

7. Mininet

  • Descripción: Un simulador enfocado en redes definidas por software (SDN) y protocolos de red avanzados.
  • Ventajas:
    • Ligero y eficiente.
    • Ideal para practicar en entornos SDN y probar controladores como OpenFlow.
  • Uso ideal: Usuarios interesados en redes de nueva generación.

8. VMware Workstation / VirtualBox

  • Descripción: Aunque son plataformas de virtualización, se pueden usar para simular redes combinando máquinas virtuales que actúan como servidores, routers o clientes.
  • Ventajas:
    • Permite crear entornos personalizados.
    • Muy flexible para prácticas de redes híbridas (físicas y virtuales).
  • Uso ideal: Diseños personalizados y pruebas con sistemas operativos.

9. Wireshark (para análisis)

  • Descripción: No es un simulador, pero se utiliza para capturar y analizar tráfico de red.
  • Ventajas:
    • Excelente para aprender sobre protocolos de red.
    • Ideal para depurar redes simuladas en Packet Tracer, GNS3, o EVE-NG.
  • Uso ideal: Análisis detallado del tráfico en redes simuladas o reales.

Conclusión:
El uso de cada herramienta depende de los objetivos y el nivel del usuario. Para principiantes, Packet Tracer es ideal, pero para proyectos más complejos, herramientas como GNS3, EVE-NG, o VIRL son opciones avanzadas. Si tu enfoque es combinar simulación con análisis, Wireshark complementa perfectamente el aprendizaje.

2. Prueba tu red

Opción 1: Crear la red físicamente o usar un simulador avanzado:

Asegúrate de que todos los dispositivos se comuniquen correctamente:

  • Comprueba que las computadoras puedan enviarse mensajes entre sí.
  • Verifica que haya acceso a internet (si es parte de tu diseño).
  • Testea que los dispositivos móviles se conecten al Wi-Fi sin problemas.

3. Documenta los resultados

Opción 1: Crear la red físicamente o usar un simulador avanzado:

Durante las pruebas, presta atención a los siguientes aspectos:

  • Problemas encontrados:
    • ¿Hubo algún error en la configuración?
    • ¿Falló algún dispositivo o conexión?
  • Soluciones aplicadas:
    • ¿Qué ajustes realizaste para solucionar los problemas?
    • Describe los pasos que seguiste.

Ejemplo:
Problema: Un punto de acceso no se conectaba a la red principal.
Solución: Reconfiguramos la dirección IP del dispositivo y revisamos las credenciales.

Opción 2: Diseñar la red usando un simulador/diagramizador básico y documentar el proceso de configuración de los componentes:

Para completar tu proyecto, es fundamental que documentes de manera detallada cómo se configura cada dispositivo de tu red. Usa la siguiente guía para estructurar tu documentación:

  1. Router o Switch:
    • Configuración inicial: Indica cómo se asignan direcciones IP a las interfaces y los nombres de los dispositivos.
    • Tablas de enrutamiento o VLANs: Describe la configuración de las VLANs y cómo se segmentan las redes.
  2. Puntos de acceso Wi-Fi:
    • Nombre de la red inalámbrica (SSID): Define el nombre visible de tu red.
    • Seguridad: Indica el tipo de encriptación utilizado (WPA2, por ejemplo) y cómo se configura.
  3. Servidores:
    • Dirección IP y servicios: Detalla las direcciones IP asignadas a cada servidor y los servicios que proporcionan (DNS, DHCP, etc.).
  4. Dispositivos finales:
    • Conexión: Explica cómo los dispositivos (PCs, impresoras, cámaras, etc.) se conectan a la red, ya sea por cable o de forma inalámbrica.

Ten en cuenta que en la opción 2, como entregable final de la simulación de red se esperan los siguientes productos:

  • Producto 1: Diagrama exportado en PDF o imagen.
  • Producto 2: Documento de configuración detallada. Incluye capturas de pantalla o fragmentos de código (si corresponde).
  • Producto 3: Breve reflexión de una página sobre los problemas encontrados y cómo los resolviste.

4. Reflexiona sobre el proceso

Opciones 1 y 2:

Al finalizar, toma un momento para analizar:

  • ¿Qué parte del diseño y montaje fue más sencilla?
  • ¿Qué desafíos enfrentaste y cómo los resolviste?
  • ¿Cómo ha cambiado tu visión sobre el papel de las redes en eventos masivos?

5. Conclusión final

Opciones 1 y 2:

Redacta un breve informe donde incluyas:

  • Los resultados de las pruebas realizadas.
  • Las mejoras que propondrías para tu diseño inicial.
  • Lo que aprendiste sobre la importancia de una red bien planificada para eventos como el Mundial.
